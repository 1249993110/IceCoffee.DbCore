存储库和服务层存在的理由
	存储库层的目的是提供CRUD操作。
	服务层的存在是为了让您能够提供返回聚合业务对象的端点, 而不是对 Web API 进行多个（昂贵的）CRUD 调用并在客户端上组装一个完整的对象。
	CRUD 对象的一个示例是产品、客户和地址。
	聚合业务对象的一个示例是发票。发票对象包含一个客户对象、两个地址对象（帐单地址、送货地址）和多个产品对象。
	服务层通过对数据库进行多次调用并将整个对象一次性发送到客户端来组装 Invoice 对象。这通过减少多个 HTTP 调用的开销和简化 API 来提高效率。
	
数据传输对象是用于封装数据并将其从应用程序的一个子系统发送到另一个子系统的对象。
	DTO 最常由 N 层应用程序中的服务层用于在自身和 UI 层之间传输数据。
	主要好处是它减少了分布式应用程序中需要通过网络发送的数据量。他们还在MVC模式中制作了出色的模型。
	DTO 的另一个用途是封装方法调用的参数。如果方法采用四个或五个以上的参数, 这可能很有用。
